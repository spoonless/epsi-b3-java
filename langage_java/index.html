Les classes du modèle de l’application

La classe Discotheque contient la liste des albums disponibles. Pour simplifier, on considère que cette classe possède une méthode charger() qui crée en dur tous les objets représentant les albums, leurs artistes et leurs pistes musicales. De plus, la classe Discotheque peut donner accès à la liste des artistes et à la liste des albums qu’elle contient.

Un artiste est représenté par la classe Artiste. Un artiste est désigné par son nom.

Un album est représenté par la classe Album. Un album est défini par :

  • son titre
  • son année de sortie
  • sa durée qui est la somme des durées des pistes musicales qu’il contient.

Une piste musicale est représentée par la classe Piste. Une piste musicale est définie par :

  • son titre
  • sa durée

En vous aidant du diagramme UML pour les relations entre classes ainsi que des indications ci-dessus, écrivez les classes Java correspondantes. Implémentez l’application de manière à ce qu’elle affiche les écrans donnés en exemple.

Important

Vous devez également utiliser une approche objet pour réaliser le reste de l’application même si aucun diagramme UML ne vous est donné.

Astuce

Pour permettre à l’utilisateur de saisir ses choix, utilisez la classe Scanner en créant une instance pour le flux d’entrée standard System.in. Vous trouverez des exemples d’utilisation de cette classe dans le chapitre sur les entrées / sorties.

  1.  Purple Haze               2:46
  2.  Manic Depression          3:46
  3.  Hey Joe (Billy Roberts)   3:23
  4.  Love or Confusion         3:15
  5.  May This Be Love          3:14
  6.  I Don't Live Today        3:55
  7.  The Wind Cries Mary       3:21
  8.  Fire                      2:34
  9.  Third Stone from the Sun  6:40
  10. Foxy Lady                 3:15
  11. Are You Experienced?      3:55

Numéro de la piste à jouer (0 pour retourner à l'accueil) :

Le comportement attendu est le suivant :

  • Si l’utilisateur choisit un numéro de piste, la validation lance la lecture de la piste.
  • Si une piste est déjà en cours de lecteur et que l’utilisateur choisit un numéro de piste, la lecture s’interrompt et la piste choisie est lue.
  • Si une piste est déjà en cours de lecture et que l’utilisateur choisit 0 (retour à l’accueil), la lecture s’interrompt et l’écran d’accueil est affiché.

Ajouter la méthode jouer() et arreter() sur la classe Piste pour lancer et arrêter la lecture du fichier son. Ainsi la classe Piste doit contenir un attribut de type File représentant le fichier qui peut être lu. Faites évoluer la classe Piste ainsi que les autres classes pour réaliser les fonctionnalités demandées.

Note

Pour lire un fichier son en Java, on peut utiliser l’API javax.sound. Ci-dessous, un exemple de code permettant de lire un fichier son avec cette API :

File file = new File("monfichier.wav");
AudioInputStream audioInputStream = AudioSystem.getAudioInputStream(file);
Clip clip = AudioSystem.getClip();
clip.open(audioInputStream);
clip.start();

Les imports nécessaires pour le code ci-dessus sont :

import java.io.File;
import javax.sound.sampled.AudioInputStream;
import javax.sound.sampled.AudioSystem;
import javax.sound.sampled.Clip;

Attention, selon votre environnement, l’API javax.sound peut ou non lire un format de fichier son. Le format WAV est normalement disponible pour tous les environnements.
